Руководство пользователя

После первого запуска программы, в папке, где лежит программа, будет создан файл настроек "config.json" со стандартными настройками и примерами заполнения полей, т.е. если этого файла нет. Если же такой файл есть, то создаваться не будет. Настройки из этого файла будут автоматический подставляться в программу.

Для начала работы необходимо:

1) Указать файлы для сравнения.
2) Убедиться, что выставленные параметры сравнения и вывода отчета
удовлетворяют вашим требованиям.
3) Нажать "Сравнить файлы".
Если в ходе работы программы возникнет ошибка, то будет выведено соответствующее сообщение.
После успешного сравнения файлов, в папке с программой появится папка "Compare_result" с отчетом, имя файла будет примерно такое: "CompareResult_1688237966862.html".
Если был выставлен признак "После сравнения открыть файл с результатами", то соответственно отчет откроется в браузере.
Если нажать на кнопку "Сохранить настройки", то все текущие настройки сравнения файлов и вывода отчета будут сохранены в файле "config.json".

Более детальное описание настройки сравнения:

Разберем на примере такого JSON объекта:

{
  "ObjectName" : "ListOKZ",
  "ObjectCaption" : "Общероссийский классификатор занятий",
  "SomeKey" : {
    "Version" : "1.5",
    "UserName" : "ShamilAbd",
    "KeyWithArray" : [
      {
        "key" : "123",
        "name" : "Руководители специализированных подразделений",
        "periodTo" : "31.12.2017 00:00:00",
        "version" : 1
      },
      {
        "key" : "777",
        "name" : "Руководители служб по сбыту, маркетингу и развитию",
        "periodTo" : null,
        "version" : 2
      }
    ]
  }
}

1) В строку "Путь до сравниваемых объектов" для этого файла необходимо указать последовательность ключей "SomeKey.KeyWithArray", через точку, т.к. в поле "SomeKey" лежит объект "KeyWithArray", а уже в нем лежит массив объектов, которые нужно сравнить.
Если такой вложенности нет, то там можно указать либо сразу ключ с объектами, либо вообще указать пустую строку, если сам файл представляет собой массив объектов для сравнения.
2) В строку "Сравниваемые ключи" необходимо в двойных кавычках перечислить список полей, которые необходимо сравнить.
В созданном "config.json" файле с примером заполнения будет идти сравнение по двум ключам: "name", "key", а "version" и "periodTo" в сравнении не будет участвовать.
3) Если выставлен признак "Сравнение с NULL всегда "не равно"", то поля с отсутствующим значением (с null) всегда будут считаться не совпавшими. Если же признак снять, то при сравнении двух null будет получен результат как совпавший. С этим признаком следует быть внимательным, т.к. если признак выставлен, и в сравниваемых полях есть поле с null, то будет много несовпавших объектов, хотя и в первом и во втором объекте это поле может быть null (условно равные), потому, возможно стоит либо отключать этот признак, либо стараться сравнивать не null поля.
4) Если выставлен признак "Сравнивать без учета регистра", то при сравнении строк "Привет", "ПрИвЕт", "привет", "ПРИВЕТ" и любые другие комбинации с регистром будут считаться совпавшими.
5) Если выставлен признак "При сравнении обрезать пробелы по краям", то при сравнении строк "Привет" и " Привет " будут считаться совпавшими, т.к. при сравнении пробелы по концам обрежутся.
6) Если выставлен признак "Найти и вывести дубликаты в файлах" то для каждого файла будет произведен поиск дублей объектов и их вывод в отчет.
7) Если выставлен признак "Вывести полностью совпавшие элементы" то в отчете будет заполнен столбец "Полностью совпавшие элементы".
8) Если выставлен признак "Вывести частично совпавшие элементы" то в отчете будет заполнен столбец "Частично совпавшие элементы".
9) Если выставлен признак "Вывести не совпавшие элементы" то в отчете будет заполнен столбец "Не совпавшие или те, для которых не осталось объектов для сравнения".
10) Если выставлен признак "Отчет только по сравниваемым ключам", то в отчете при выводе объекта в поля будут использованы лишь сравниваемые ключи, а не все ключи объекта. Тут проще запустить два раза отчет, с выставленным и снятым признаком, и посмотреть самому.
11) Если выставлен признак "Добавить номера объектам", то к каждому объекту в своей колонке будет присвоен порядковый номер.
12) Если выставлен признак "Добавить запятые между объектами", то между объектами будут добавлены запятые. Удобно, если нужно скопировать результат в какой-то JSON файл и сделать из них массив объектов.
13) Параметр "Пробелов в объекте" указывает, сколько отступов будет добавлено на каждый уровень объекта. Как в примере, до "ObjectName" и "ObjectCaption" идет 2 отступа, а "Version" и "UserName" уже следующий уровень, т.е. там уже будет 4 отступа. По умолчанию количество отступов равно двум.
